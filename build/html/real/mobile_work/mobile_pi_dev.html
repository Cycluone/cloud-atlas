移动开发:树莓派开发环境构建¶

我在最初的 移动工作思考 方案中，是想采用 macOS 的 MacBook Pro来实现的: 我从2011年自费购买了MacBook Air 11” 就一直使用 Apple Atlas 家的系列产品，确实是非常成熟的软硬件融合的生态。不过，2022年3月，魔都一个多月疫情封闭在家，我开始折腾 树莓派Raspberry Pi 400 运行 sway - i3兼容Wayland compositor (基于 wayland显示服务器协议 的平铺式窗口管理器)，意外发现 wayland+sway 居然能够充分释放 Raspberry Pi 的性能，这触发我思考用日常 Linux桌面 来完成所有开发运维，磨炼技术的方案。

Raspberry Pi OS¶

我在 Raspberry Pi 上使用过多种Linux发行版:

  • Ubuntu Linux : 部署ARM架构Kubernetes

  • Alpine Linux : 在树莓派上部署K3s

  • Kali Linux : 树莓派安装Kali Linux 并 运行sway窗口管理器

不过，上述第三方Linux for Raspberry Pi 各自有一些限制，对于充分发挥树莓派性能以及特有功能存在不足。所以最终，作为图形桌面，我还是回归到树莓派官方提供的 Raspberry Pi OS :

  • 官方 Raspberry Pi OS 提供了针对树莓派硬件配置的 raspi-config 工具，可以非常方便调整系统设置

  • 官方 Raspberry Pi OS 升级包含了firmware升级，可以为系统带来更多改进

  • 出现和树莓派硬件相关的问题更容易找到参考资料

安装Raspberry Pi OS¶

  • 安装过程非常简单，以下仅记录一些特定点

  • 修订时区:

配置上海本地时区¶
cd /etc
sudo unlink localtime
sudo ln -s /usr/share/zoneinfo/Asia/Shanghai localtime

  • 更新系统:

APT更新整个系统¶
sudo apt update
sudo apt upgrade -y

  • 安装 sway - i3兼容Wayland compositor 图形管理器:

    sudo apt install sway

安装完成后，执行 sway 就可以进入桌面，不过为了能够更为方便，修订 ~/.zshrc 添加以下内容方便自动启动 sway :

复制sway个人配置¶
if [ -z $DISPLAY ] && [ "$(tty)" = "/dev/tty1" ]; then
  exec sway
fi

  • 安装应用软件、中文字体和输入法
