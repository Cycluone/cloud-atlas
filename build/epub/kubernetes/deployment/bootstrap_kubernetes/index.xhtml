Kubernetes集群引导

这个章节内容是我实践真实部署的记录，不过由于kubernetes有多种部署模式，所以我这里概述如下：

  • 创建单一控制平面(单master)集群 依然是单节点部署，有点类似 安装和运行minikube ，所以仅作为个人学习和开发，不适合生产部署

  • 真实的部署生产环境，则应该按照 高可用(HA)Kubernetes集群 模式进行部署：

    • 首先部署 基于负载均衡的高可用Kubernetes集群

      • 我也实践了通过DNSRR+HAProxy实现负载均衡(因为环境所限没有可用VIP)

    • 然后在负载均衡环境中，就可以 使用kubeadm创建高可用集群

    • 然后进一步可以拆分etcd，完成 高可用etcd集群部署 ，就构建 扩展etcd节点高可用集群 ，可以实现企业级Kubernetes
